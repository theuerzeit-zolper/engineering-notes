# Engineering Notes

Eine strukturierte Sammlung von Architekturentscheidungen, Infrastrukturdesigns, Identitäts- und Zugriffssystemen sowie technischen Erkenntnissen aus realen Projekten.

Dieses Repository ist keine Sammlung von Anleitungen.

Stattdessen dokumentiert es, wie Systeme entworfen wurden, warum bestimmte Entscheidungen getroffen wurden und welche Kompromisse dabei bewusst eingegangen wurden.

---

# Einstieg

Die dokumentierten Projekte befinden sich im Verzeichnis:

→ [architecture](architecture)

Jedes Projekt beschreibt eine konkrete Architekturentscheidung oder Systemlösung und verweist bei Bedarf auf weiterführende Dokumente aus den Bereichen Infrastruktur, Identity und Betriebserfahrungen.

# Grundsätze

* Verständlichkeit vor Komplexität
* Praxis vor Theorie
* Entscheidungen statt Produktbeschreibungen
* Systeme statt Einzelkomponenten
* Nachvollziehbarkeit vor Blackbox-Lösungen
